# Requisitos Funcionais e Não Funcionais
### H1: Marcação de Consultas Online  
Como paciente, eu quero marcar consultas online para evitar deslocamentos desnecessários.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes marquem consultas online.  
- Requisitos Não Funcionais:  
  1.O sistema deve permitir que os pacientes naveguem sem dificuldades.
  2. O sistema deve responder rapidamente às ações dos usuários durante a marcação de consultas.  

### H2: Envio de Exames  
Como paciente, eu quero enviar meus exames pelo aplicativo para que o médico os tenha antes da consulta.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes enviem exames pelo aplicativo.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a segurança das informações pessoais durante o envio de exames.  
  2. O sistema deve processar o envio de exames de forma eficiente e rápida.  

### H3: Visualização de Prontuário  
Como paciente, eu quero visualizar meu prontuário online para acompanhar meu histórico médico.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes visualizem seu prontuário online.  
- Requisitos Não Funcionais:  
  1. O sistema deve ser acessível em diferentes dispositivos, como smartphones e computadores.  
  2. O sistema deve garantir que o acesso ao prontuário seja rápido e responsivo.  

### H4: Fila de Atendimento  
Como paciente, eu quero acompanhar a fila de atendimento para saber quando serei atendido.  
- Requisitos Funcionais:  
  1. O sistema deve exibir a posição do paciente na fila de atendimento.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir que a exibição da fila seja clara e atualizada em tempo real.  
  2. O sistema deve ter um desempenho ágil para processar informações de fila rapidamente.  

### H5: Gerenciamento de Horários pelo Médico  
Como médico, eu quero gerenciar meus horários de consulta para organizar minha agenda.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os médicos gerenciem seus horários de consulta.  
- Requisitos Não Funcionais:  
  1. O sistema deve ser intuitivo para que os médicos possam gerenciar suas agendas com facilidade.  
  2. O sistema deve garantir que as atualizações de horário sejam salvas de forma rápida.  

### H6: Histórico de Consultas dos Pacientes  
Como médico, eu quero acessar o histórico de consultas dos pacientes para fornecer um atendimento melhor.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os médicos acessem o histórico de consultas dos pacientes.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a segurança dos dados dos pacientes durante o acesso ao histórico.  
  2. O sistema deve ser capaz de processar informações rapidamente para melhorar o atendimento.  

### H7: Comunicação Segura entre Pacientes e Médicos  
Como paciente, eu quero enviar mensagens seguras ao meu médico para tirar dúvidas sobre meu tratamento.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes enviem mensagens seguras ao médico.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a criptografia das mensagens para segurança das comunicações.  
  2. O sistema deve assegurar que as mensagens sejam entregues rapidamente.  

### H8: Realização de Teleconsultas  
Como paciente, eu quero fazer consultas por vídeo para evitar deslocamentos.  
- Requisitos Funcionais:  
  1. O sistema deve suportar consultas por vídeo.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir uma qualidade de videochamada mínima de HD (720p).  
  2. O sistema deve ser responsivo e estável durante as teleconsultas.  

### H9: Receitas Médicas Digitais  
Como paciente, eu quero receber receitas médicas digitais para facilitar a compra de medicamentos.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os médicos enviem receitas médicas digitais para os pacientes.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a segurança e integridade das receitas digitais.  
  2. O sistema deve ser acessível para que os pacientes possam visualizar suas receitas facilmente.  

### H10: Notificações e Lembretes de Consulta  
Como paciente, eu quero receber lembretes de minhas consultas para não esquecer meus compromissos.  
- Requisitos Funcionais:  
  1. O sistema deve enviar lembretes de consultas para os pacientes.  
- Requisitos Não Funcionais:  
  1. O sistema deve enviar notificações de forma eficiente e em tempo hábil.  
  2. O sistema deve permitir que os usuários escolham o método de notificação preferido (e-mail ou push notification).  

### H11: Pagamento de Consultas Online  
Como paciente, eu quero pagar minhas consultas online.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes paguem suas consultas online.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a segurança das transações financeiras durante o processo de pagamento.  
  2. O sistema deve processar pagamentos de forma rápida e eficiente.  

### H12: Feedback de Atendimento  
Como paciente, eu quero deixar feedback sobre meu atendimento para ajudar a melhorar o serviço.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes deixem feedback sobre o atendimento recebido.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a confidencialidade dos feedbacks enviados.  
  2. O sistema deve ser fácil de usar, permitindo que os pacientes deixem feedback rapidamente.  

### H13: Histórico de Pagamentos  
Como paciente, eu quero visualizar meu histórico de pagamentos para controle financeiro.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes visualizem seu histórico de pagamentos.  
- Requisitos Não Funcionais:  
  1. O sistema deve ser acessível e fácil de navegar para que os pacientes encontrem rapidamente seu histórico.  
  2. O sistema deve garantir a segurança das informações financeiras exibidas.  

### H14: Gerenciamento de Perfil do Paciente  
Como paciente, eu quero gerenciar meu perfil e informações de contato para mantê-las atualizadas.  
- Requisitos Funcionais:  
  1. O sistema deve permitir que os pacientes editem suas informações pessoais e de contato.  
- Requisitos Não Funcionais:  
  1. O sistema deve garantir a segurança das informações pessoais durante o processo de edição.  
  2. O sistema deve ser intuitivo, facilitando a atualização das informações pelo paciente.
